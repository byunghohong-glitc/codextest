# Repository Guidelines

## 작업 환경
- 운영체제: Windows_NT
- Shell: PowerShell 5.1.19041.6456
- 프로젝트 루트: `C:\github\codextest`
- 주요 런타임: `node v24.15.0`, `git 2.54.0.windows.1`, `python UNKNOWN`, `npm UNKNOWN`

## 작업 범위
이 저장소에서는 `C:\github\codextest` 아래의 파일만 읽고 수정한다. 프로젝트 밖 파일은 다루지 않는다.
`legacy/`와 `generated/`는 요청받지 않으면 수정하지 않는다.

## 기본 원칙
1. 프로젝트 내부 파일을 직접 읽고 판단한다.
2. 추정으로 구현하지 않는다.
3. 요구사항이 모호하면 짧게 확인하거나, 확인이 어려우면 안전한 최소 범위로 작업한다.
4. 필요한 경우 먼저 짧은 계획을 세운 뒤 작업한다.
5. TypeScript 파일을 수정한 뒤에는 `pnpm typecheck`와 관련 unit test를 실행한다.
6. API response field name은 하위 호환성을 위해 임의로 변경하지 않는다.
7. 코드 수정 후 관련 테스트 또는 최소 검증을 수행한다.
8. `관련 unit test`는 수정한 모듈을 직접 검증하는 테스트를 뜻한다.
9. `최소 검증`은 빌드, 타입체크, 대상 테스트 중 가능한 가장 작은 확인 절차를 뜻한다.
10. 테스트를 실행할 수 없으면 그 이유를 명확히 기록한다.
11. 변경 내용과 검증 결과를 마지막에 짧게 요약한다.
12. `.env`, secret, credential, API key, 인증 파일은 출력하거나 수정하지 않는다.
13. 삭제, 강제 덮어쓰기, 원격 push 같은 파괴적 작업은 수행하지 않는다.
14. 사용자가 제공한 변수명, 함수명, 파일명은 임의로 바꾸지 않는다.
15. 불필요한 대규모 리팩토링은 하지 않는다.
16. 요청 범위 밖의 기능은 추가하지 않는다.

## 명령과 검증
- 현재 저장소에는 고정된 빌드/테스트 명령이 없다.
- TypeScript 변경이 생기면 `pnpm typecheck`를 우선 실행한다.
- 다른 변경은 영향 범위에 맞는 최소 검증만 수행한다.
- 검증 명령이 추가되면 이 섹션을 함께 갱신한다.

## 스타일
- 기본 문서는 한국어로 작성한다.
- 새 파일명은 역할이 드러나도록 짧고 명확하게 짓는다.
- 포맷터나 린터가 도입되면 사용 규칙을 이 파일에 추가한다.

## 협업
- 이후 Codex의 설명, 계획, 보고는 기본적으로 한국어로 작성한다.
- 변경 전에는 관련 파일을 직접 읽고 범위를 확인한다.
- 완료 후에는 생성한 파일 목록만 짧게 보고하고, `AGENTS.md` 전체 내용을 반복하지 않는다.
- PR에는 변경 요약, 검증 결과, 영향 범위를 적는다.
